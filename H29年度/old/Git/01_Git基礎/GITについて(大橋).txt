■Gitとは

git（ギット）は、プログラムのソースコードなどの変更履歴を記録・追跡するための分散型バージョン管理システムのこと

変更履歴を管理したいディレクトリをリポジトリの管理下に置くことで、
そのディレクトリ内のファイルやディレクトリの変更履歴を記録することができる。

◆リポジトリとは

ファイルやディレクトリの状態を記録する場所。
保存された状態は、内容の変更履歴として格納されている。

・リモートリポジトリとは

専用のサーバに配置して複数人で共有するためのリポジトリです。

・ローカルリポジトリとは

ユーザ一人ひとりが利用するために、自分の手元のマシン上に配置するリポジトリです。


◆メリット

プログラムのソースコードなどがバージョン管理できる。

デグレが発生する可能性が低くなる。（品質の向上）

作業効率が上がる。（パフォーマンスの向上）


◆デメリット

コスト（費用・導入コスト・Gitの学習コスト）がかかる

※GitHubを使う場合、プライベートリポジトリは有料

※Gitの学習コストはSVNよりも高い（Git導入でのトラブル＝ギットラブルという造語が作られるくらい）


◆SVNとの違いは？

・Gitではローカルコミットができる、（SVNはできない）

リポジトリに影響を与えずに、ローカルで作業単位のバージョン管理ができる。


・Gitではブランチが簡単に切れる

SVNでは「親ブランチ←子ブランチ」のみマージできる

Gitでは「子ブランチ←孫ブランチ」もマージできる


・GitではGitHubが利用できる（SVNはできない）

◆GitHubとは

ソフトウェア開発プロジェクトのためのソースコード管理サービス

公開されているソースコードの閲覧や簡単なバグ管理機能、SNSの機能

■使用方法

１．基本操作

◆ローカルリポジトリ

・新規リポジトリ作成（Git ここにリポジトリを作成）

・コミット（Commit）

ローカルリポジトリに変更を保存する


◆リモートリポジトリ

・クローン（Clone）

クローンを実行すると、リモートリポジトリの内容をまるまるダウンロードしてきて、
別のマシンにローカルリポジトリとして作成できます。

※SVNのチェックアウトと同義

・プッシュ（Push）

ローカルリポジトリの変更内容をリモートリポジトリにアップロードする

・プル（Pull）

リモートリポジトリからローカルリポジトリを更新する
