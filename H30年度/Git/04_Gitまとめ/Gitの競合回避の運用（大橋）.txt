EXCELやWordファイルが競合(コンフリクト)するのを回避する運用


１．管理者が構成管理する

プロジェクト内で、構成管理者をたてる。
メンバーは基本的に構成管理者にレビュー依頼（GitHubであればプルリクエスト）を投げる。
（その際、コメントはわかりやすく書く）

構成管理者がGitにプッシュする
競合する場合は、構成管理者が判断してマージする。

メリット：メンバーが、競合を気にせずに作業することができる
デメリット：構成管理者をたてる必要があり、コストがかかる



２．競合する可能性のあるファイルをサーバーで共有する

競合する可能性のあるファイルは、ある程度事前にわかっているはず。
（共通部品や設定ファイルなど）

それらのファイルは、サーバーの特定フォルダに置き、
そのファイルをメンバー全員で修正する運用とする。
（一日の始めにバックアップをとる運用とする）

定期的に、構成管理者がGitにコミットする。

メリット：ファイルの編集中はロックされるため、競合がおきない。
デメリット：Gitを使わないため、バージョン管理が雑になる。
